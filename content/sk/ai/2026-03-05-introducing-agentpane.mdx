# Predstavujeme agentpane

agentpane je lokálne webové používateľské rozhranie pre AI agentov na programovanie. Spustíte `npx agentpane`, otvorí sa v prehliadači a môžete komunikovať s Claude Code, Codexom alebo akýmkoľvek agentom, ktorý podporuje [ACP (Agent Client Protocol)](https://github.com/anthropics/agent-protocol) — a to všetko vo viacpanelovom rozhraní so streamovaním, perzistenciou a správou relácií:

```bash
npx agentpane
# → http://localhost:6767
```

Vyberte agenta, vyberte pracovný adresár a spustite reláciu. Agent beží ako podproces na vašom počítači. Žiadny cloud, žiadne nasadzovanie, žiadne účty.

![Obrazovka nastavenia agentpane s výberom agenta a adresára](/blog-images/agentpane-setup.png)


## Viac panelov, viac agentov

Najviac som chcel možnosť spustiť naraz viacerých agentov a vidieť ich vedľa seba. agentpane vám ponúka až štyri panely s nastaviteľnou veľkosťou a kartami. Relácie môžete presúvať z bočného panela do ľubovoľného panela alebo potiahnutím karty vytvoriť nový rozdelený panel. Rozloženie zostane zachované aj po obnovení stránky.

To znamená, že v jednom paneli môžete mať Claude Code, ktorý pracuje na vašom backende, a v druhom Codex, ktorý refaktoruje váš frontend — na tej istej obrazovke, v tej istej aplikácii.

![Dve relácie bežia vedľa seba — Codex vľavo, Claude Code vpravo](/blog-images/agentpane-hero.png)

## Architektúra

Aplikácia beží ako dva procesy:

* **API server** (Hono, port 3456) — spúšťa agentov, spravuje relácie a zabezpečuje trvalé ukladanie dát
* **Webový frontend** (Next.js, port 6767) — používateľské rozhranie, pričom volania `/api` sú presmerované na backend

Backend je postavený na [Effect.ts](https://effect.website/) a používa vkladanie závislostí prostredníctvom skladateľných servisných vrstiev:

```
SessionRepo (DB)
  → ConnectionManager (agent subprocess lifecycle)
  → PromptEngine (turn orchestration)
  → EventHub / EventBroadcaster (SSE)
  → WriteQueue (batched DB writes)
```

Agenti komunikujú cez ACP — JSON-RPC 2.0 cez stdio. Keď odošlete prompt, `PromptEngine` vytvorí záznamy jednotlivých krokov, odošle správu podprocesu agenta cez ACP a priebežne posiela výsledky späť prostredníctvom Server-Sent Events.


### Streamovanie s bezpečným opätovným pripojením

Frontend sa pre každú reláciu pripája k SSE endpointu. Každá udalosť dostane monotónne ID. `EventBroadcaster` udržiava kruhový buffer (512 KB / 1000 udalostí), takže ak sa váš prehliadač odpojí a znova pripojí, prehrá zmeškané udalosti pomocou hlavičky `Last-Event-ID`. Žiadne medzery v konverzácii, žiadne manuálne obnovovanie.

### Perzistencia odolná proti pádom

Všetok stav sa ukladá do lokálnej databázy SQLite v `~/.agentpane/agentpane.db`. Zápisy sa však nevykonávajú priamo — `WriteQueue` zhromažďuje operácie a každých 50 ms ich hromadne zapíše. Pred samotným zápisom sa operácie uložia do obnovovacej tabuľky. Ak server spadne uprostred zápisu, po reštarte pokračuje tam, kde skončil. Bez straty správ.

```sql
-- Tabuľka obnovy
CREATE TABLE write_queue_ops (
  id TEXT PRIMARY KEY,
  session_id TEXT NOT NULL,
  op_json TEXT NOT NULL,
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);
```


## Čo agenti vidia

agentpane nerieši, ktorý model agenta poháňa — komunikuje cez ACP. Keď sa pripojíte k relácii, `ConnectionManager`:

1. Nájde binárku agenta (prejde `node_modules/.bin/` kvôli kompatibilite s npm)
2. Spustí ju ako podproces so stdio pipe-mi
3. Dohodne ACP capabilities (auth, config, modes, commands)
4. Pokúsi sa obnoviť reláciu, ak sa znovu pripája k predchádzajúcej relácii

Používateľské rozhranie sa prispôsobí tomu, čo agent podporuje. Ak agent sprístupňuje možnosti konfigurácie, zobrazia sa ako rozbaľovacie ponuky. Ak podporuje slash príkazy, zobrazí sa automatické dopĺňanie. Ak streamuje bloky úvah, zobrazia sa ako zbaliteľné sekcie.

## Frontend

Webová aplikácia beží na Next.js 16 s Reactom 19 a React Compilerom — nikde netreba manuálne používať `useMemo` ani `useCallback`. Stav je rozdelený do dvoch vrstiev kontextu:

* **SessionProvider** — aktívna relácia, kontroly stavu, CRUD operácie relácie
* **LayoutProvider** — konfigurácia panelov, poradie kariet, drag-and-drop

Stav na serveri (relácie, konverzácie, využitie tokenov) sa spravuje pomocou TanStack React Query a pri SSE udalostiach sa invaliduje. Markdown sa vykresľuje priebežne počas streamovania pomocou [Streamdown](https://github.com/anthropics/streamdown) so zvýrazňovaním syntaxe cez Shiki.
