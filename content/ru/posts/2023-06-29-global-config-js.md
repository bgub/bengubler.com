Современная веб-разработка предполагает работу с множеством JS-инструментов сборки и фреймворков, и каждому из них нужны собственные конфигурационные файлы. Управление этими конфигурационными файлами, такими как `.eslintrc`, `next.config.js` и `tailwind.config.js`, может быть утомительным и отнимать много времени. В этой публикации я рассмотрю идею объединить эти конфигурационные файлы в один — `global.config.js`, чтобы централизовать конфигурацию проекта и уменьшить количество лишних отвлечений.

## Конфигурационные файлы повсюду

Пока я пишу эту публикацию, в корне моего проекта лежат `.eslintrc.json`, `next.config.js`, `postcss.config.js`, `tailwind.config.js` и `tsconfig.json`. Хотя у меня довольно стандартная конфигурация и каждый файл занимает меньше 30 строк, они отнимают ценное место в боковой панели VSCode и отвлекают от того, что действительно важно: исходного кода.

Мой случай далеко не уникален. Во многих других проектах конфигурационных файлов ещё больше. Представьте, насколько захламлённым может стать проект, если добавить `.babelrc`, `prettier.config.js`, `jest.config.js`, `cypress.json` и т. д.

## Решение: `global.config.js`

Я хотел бы предложить простое решение: объединить конфигурационные файлы в одном файле под названием `global.config.js`. Конфигурация каждого инструмента хранилась бы в экспортируемом объекте под ключом с именем npm-пакета.

Проекты по-прежнему должны поддерживать использование отдельных конфигурационных файлов в случаях, когда конфигурация большая и сложная (например, `tsconfig.json`), но сначала им следует проверять, существует ли `global.config.js` и есть ли в нём конфигурация для их инструмента.

Вот как может выглядеть простой `global.config.js`:

```js
module.exports = {
  eslint: {
    extends: ["next/core-web-vitals"]
  },
  postcss: {
    plugins: {
      tailwindcss: {},
      autoprefixer: {}
    }
  },
  tailwindcss: {
    content: ["./src/pages/**/*.{js,ts,jsx,tsx,mdx}"],
    theme: {
      extend: {
        backgroundImage: {
          "gradient-radial": "radial-gradient(var(--tw-gradient-stops))"
        }
      }
    },
    plugins: [require("@tailwindcss/typography")]
  }
}
```

## Но что насчёт типов?

Автодополнение типов для `global.config.js` можно легко включить, добавив определения типов в комментариях — так же, как это уже делают Tailwind и NextJS в своих конфигурационных файлах.

```js
module.exports = {
  // ...

  /** @type {import('tailwindcss').Config} */
  tailwindcss: {
    content: ["./src/pages/**/*.{js,ts,jsx,tsx,mdx}"],
    theme: {
      extend: {
        backgroundImage: {
          "gradient-radial": "radial-gradient(var(--tw-gradient-stops))"
        }
      }
    },
    plugins: [require("@tailwindcss/typography")]
  }
}
```
