Современная веб-разработка предполагает работу с несколькими JS-инструментами для сборки и фреймворками, каждому из которых нужны собственные конфигурационные файлы. Управление такими файлами, как `.eslintrc`, `next.config.js` и `tailwind.config.js`, может быть утомительным и отнимать много времени. В этой публикации я рассмотрю идею объединить все эти конфигурационные файлы в один — `global.config.js`, чтобы собрать конфигурацию проекта в одном месте и уменьшить количество отвлекающих мелочей.

## Конфигурационные файлы повсюду

Пока я пишу эту публикацию, в корне моего проекта лежат `.eslintrc.json`, `next.config.js`, `postcss.config.js`, `tailwind.config.js` и `tsconfig.json`. Хотя у меня почти всё настроено «из коробки» и каждый файл занимает меньше 30 строк, эти файлы съедают ценное место в боковой панели VSCode и отвлекают от того, что действительно важно: исходного кода.

Мой случай вовсе не исключение. В некоторых проектах конфигурационных файлов гораздо больше. Представьте, насколько захламлённым может стать проект, если добавить `.babelrc`, `prettier.config.js`, `jest.config.js`, `cypress.json` и т. д.

## Решение: `global.config.js`

Я хочу предложить простое решение: объединить все конфигурационные файлы в одном файле под названием `global.config.js`. Конфигурация для каждого инструмента будет храниться в экспортируемом объекте под ключом с именем npm-пакета.

Проекты по-прежнему должны поддерживать использование отдельных конфигурационных файлов в случаях, когда конфигурация объёмная и сложная (например, `tsconfig.json`), но сначала они должны проверять, существует ли `global.config.js` и есть ли в нём конфигурация для их инструмента.

Вот как может выглядеть простой `global.config.js`:

```js
module.exports = {
  eslint: {
    extends: ["next/core-web-vitals"]
  },
  postcss: {
    plugins: {
      tailwindcss: {},
      autoprefixer: {}
    }
  },
  tailwindcss: {
    content: ["./src/pages/**/*.{js,ts,jsx,tsx,mdx}"],
    theme: {
      extend: {
        backgroundImage: {
          "gradient-radial": "radial-gradient(var(--tw-gradient-stops))"
        }
      }
    },
    plugins: [require("@tailwindcss/typography")]
  }
}
```

## А что насчёт типов?

Автодополнение типов для `global.config.js` можно без труда включить, добавив определения типов в комментариях — как это уже делают Tailwind и NextJS в своих конфигурационных файлах.

```js
module.exports = {
  // ...

  /** @type {import('tailwindcss').Config} */
  tailwindcss: {
    content: ["./src/pages/**/*.{js,ts,jsx,tsx,mdx}"],
    theme: {
      extend: {
        backgroundImage: {
          "gradient-radial": "radial-gradient(var(--tw-gradient-stops))"
        }
      }
    },
    plugins: [require("@tailwindcss/typography")]
  }
}
```
