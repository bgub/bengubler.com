В лаборатории, где я работаю, у нас есть доступ к среде высокопроизводительных вычислений (HPC), в которой используется [Slurm Workload Manager](https://slurm.schedmd.com/documentation.html).

Я пользуюсь этой системой уже некоторое время и нашёл несколько команд, которыми постоянно пользуюсь. Решил поделиться ими здесь — вдруг они окажутся полезны кому-то ещё.

## Проверка статуса задач

```bash
# Просмотр всех задач
squeue
# Проверка статуса только ваших задач
squeue -u <username>
# Проверка статуса задач с определённым QOS
squeue -q <QOS>
```

## Отмена задач

```bash
# Отменить конкретную задачу
scancel <job_id>
# Отменить все свои задачи
scancel -u <username>
```

## Запрос узла в интерактивном режиме

```bash
# Запрос одного узла (откроется интерактивный сеанс в вашем терминале)
# После выхода из терминала узел будет освобождён
salloc --nodes=1 --gpus=8 --qos=<QOS> --mem=2000G --time=72:00:00 --ntasks=1 --cpus-per-task=128
```

## Отправка задачи

Что делать, если все вычислительные узлы уже заняты или вы не хотите, чтобы задача завершалась сразу после закрытия терминала? В таком случае можно воспользоваться `sbatch`, чтобы отправить задачу в очередь. Она автоматически запустится, как только для неё удастся выделить ресурсы.

Здесь потребуется чуть больше подготовки. Предположим, что задача, которую мы хотим запустить, находится в `myjob.sh`. Чтобы отправить этот скрипт как задачу, сначала создадим Bash-скрипт, который будет запускать Slurm. Назовём его `run.sh`:

```bash
#!/bin/bash
#SBATCH -J "JOBNAME"
#SBATCH --nodes=1
#SBATCH --gpus-per-node=8
#SBATCH --cpus-per-task=128
#SBATCH --mem=2000G
#SBATCH --time=72:00:00
#SBATCH --qos=<QOS>

srun --nodes=1 myjob.sh
```

Обратите внимание, что мы используем директиву `#SBATCH`, чтобы передать параметры, которые раньше передавали в `salloc`. Мы также используем `srun`, чтобы запустить саму задачу; эта команда обеспечит запуск скрипта на нескольких узлах, если это потребуется.

Наконец, чтобы запустить наш скрипт, выполним:

```bash
sbatch run.sh
```
