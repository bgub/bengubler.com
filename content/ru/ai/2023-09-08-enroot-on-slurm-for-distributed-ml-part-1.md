В лаборатории, где я работаю, у нас есть доступ к среде High Performance Computing (HPC), в которой используется [Slurm Workload Manager](https://slurm.schedmd.com/documentation.html). Наш HPC работает на RHEL (Red Hat Enterprise Linux) 7, а пользовательские разрешения существенно ограничены. У нас нет доступа к `sudo`, и с вычислительных узлов нельзя выйти в интернет.

Более того, установка пакетов и обновление драйверов непосредственно на вычислительном узле настолько затруднены, что распределённое обучение с использованием современного ПО становится невероятно сложной задачей. К счастью, решение есть: контейнеризация.

Мы будем использовать Docker, чтобы собрать образ на локальной машине, в котором будут все нужные нам пакеты. Затем мы сможем перенести этот образ в HPC и использовать его для запуска нашего скрипта обучения.

## Шаг 1: Соберите Docker-образ локально

Я уже писал о [конфигурации Docker, которую использую для machine learning](./ultimate-ml-dockerfile), так что здесь повторяться не буду. Главное, чтобы у вас был Docker-образ с тегом и пакетами, необходимыми для запуска скрипта обучения. У меня используется Ubuntu 20.04 и CUDA 11.8.

## Шаг 2: Сожмите и перенесите образ

Установите [Enroot](https://github.com/NVIDIA/enroot), а затем выполните следующую команду, чтобы преобразовать Docker-образ в файл squashfs:

```bash
enroot import dockerd://<image-name>
```

Это создаст файл с именем `<image-name>.sqsh` в текущем каталоге. Скопируйте этот файл на HPC с помощью `scp`.

## Шаг 3: Загрузите образ на HPC

Подключитесь к вычислительному узлу с помощью `salloc --nodes=1 --gpus=8 --qos=<qos> --mem=2000G --time=72:00:00 --ntasks=1 --cpus-per-task=128`.

Сначала нужно загрузить модуль Enroot:

```bash
module load jq zstd pigz parallel libnvidia-container enroot
```

На HPC создайте образ следующей командой:

```bash
enroot create --name image-name /path/to/image-name.sqsh
```

После этого запустите его:

```bash
enroot start --mount /local/file/path:/image/file/path \
             --rw image-name bash
```

Это откроет интерактивную оболочку внутри контейнера. Не забудьте указать флаг `--rw`, который делает корневую файловую систему доступной для записи. Вы можете добавить столько флагов `--mount`, сколько нужно, чтобы смонтировать файлы и каталоги с хост-машины.

Если вы хотите передать переменные окружения, можно использовать флаг `--env` вместе с именем переменной окружения на хост-машине. Например, `--env SLURM_NODEID` передаст переменную окружения `SLURM_NODEID`.
