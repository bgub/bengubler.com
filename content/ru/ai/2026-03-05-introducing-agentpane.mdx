# Знакомьтесь: agentpane

agentpane — это локальный веб-интерфейс для ИИ-агентов, помогающих писать код. Вы запускаете `npx agentpane`, интерфейс открывается в браузере, и вы можете общаться с Claude Code, Codex или любым агентом, поддерживающим [ACP (Agent Client Protocol)](https://github.com/anthropics/agent-protocol), — и всё это в интерфейсе с несколькими панелями, стримингом, сохранением состояния и управлением сессиями:

```bash
npx agentpane
# → http://localhost:6767
```

Выберите агента, укажите рабочую директорию и запустите сессию. Агент работает как подпроцесс на вашем компьютере. Без облака, развёртывания и аккаунтов.

## Несколько панелей, несколько агентов

Главное, чего я хотел, — это возможность запускать сразу несколько агентов и видеть их рядом. agentpane позволяет открыть до четырёх панелей с вкладками и менять их размер. Можно перетаскивать сессии из боковой панели в любую из них, а можно перетащить вкладку, чтобы открыть её в новой панели. Раскладка сохраняется даже после обновления страницы.

Это значит, что в одной панели Claude Code может работать над вашим бэкендом, а в другой Codex — рефакторить фронтенд: один экран, одно приложение.

![Две сессии бок о бок — Codex слева, Claude Code справа](/blog-images/agentpane-hero.png)

## Архитектура

Приложение состоит из двух процессов:

* **API-сервер** (Hono, порт 3456) — запускает агентов, управляет сессиями и отвечает за сохранение данных
* **Веб-фронтенд** (Next.js, порт 6767) — пользовательский интерфейс, в котором вызовы `/api` перенаправляются на бэкенд

Бэкенд построен на [Effect.ts](https://effect.website/) и использует внедрение зависимостей через компонуемые сервисные слои:

```
SessionRepo (DB)
  → ConnectionManager (agent subprocess lifecycle)
  → PromptEngine (turn orchestration)
  → EventHub / EventBroadcaster (SSE)
  → WriteQueue (batched DB writes)
```

Агенты взаимодействуют через ACP — JSON-RPC 2.0 поверх stdio. Когда вы отправляете промпт, `PromptEngine` создает записи шагов диалога, передает сообщение подпроцессу агента через ACP и потоково возвращает результаты через Server-Sent Events.

### Стриминг с безопасным переподключением

Фронтенд подписывается на SSE-эндпоинт для каждой сессии. Каждому событию присваивается монотонно возрастающий ID. `EventBroadcaster` хранит кольцевой буфер (512KB / 1000 событий), поэтому, если браузер отключится, а затем переподключится, пропущенные события будут воспроизведены с помощью заголовка `Last-Event-ID`. Никаких пробелов в диалоге и никаких ручных обновлений.

### Отказоустойчивое сохранение

Всё состояние сохраняется в локальную базу данных SQLite по пути `~/.agentpane/agentpane.db`. Но запись происходит не сразу — `WriteQueue` накапливает операции и пакетно сбрасывает их каждые 50 мс. Перед сбросом операции сохраняются в таблице восстановления. Если сервер аварийно завершится во время сброса, после перезапуска он продолжит с того места, на котором остановился. Сообщения не теряются.

```sql
-- Таблица восстановления
CREATE TABLE write_queue_ops (
  id TEXT PRIMARY KEY,
  session_id TEXT NOT NULL,
  op_json TEXT NOT NULL,
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);
```

## Что видят агенты

Для agentpane не важно, какая модель лежит в основе агента — он взаимодействует по ACP. Когда вы подключаетесь к сессии, `ConnectionManager`:

1. Находит исполняемый файл агента (просматривает `node_modules/.bin/` для совместимости с npm)
2. Запускает его как дочерний процесс с каналами stdio
3. Согласовывает возможности ACP (auth, config, modes, commands)
4. Пытается возобновить сессию при повторном подключении к предыдущей сессии

Интерфейс подстраивается под возможности агента. Если агент предоставляет параметры конфигурации, они отображаются в виде выпадающих списков. Если он поддерживает slash-команды, вы получаете автодополнение. Если он передаёт блоки мыслей потоком, они отображаются как сворачиваемые секции.

## Фронтенд

Веб-приложение построено на Next.js 16, React 19 и React Compiler — `useMemo` и `useCallback` нигде не используются вручную. Состояние разделено на два слоя контекста:

* **SessionProvider** — активная сессия, проверки работоспособности, создание, чтение, обновление и удаление сессий
* **LayoutProvider** — конфигурация панелей, порядок вкладок, перетаскивание

Серверное состояние (сессии, диалоги, использование токенов) управляется через TanStack React Query и инвалидируется по событиям SSE. Markdown рендерится прямо по мере поступления через [Streamdown](https://github.com/anthropics/streamdown) с подсветкой синтаксиса от Shiki.
