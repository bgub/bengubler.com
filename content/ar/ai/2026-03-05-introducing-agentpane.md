# التعرّف على agentpane

agentpane هي واجهة ويب محلية لوكلاء البرمجة بالذكاء الاصطناعي. شغّل `npx agentpane`، فيُفتح في متصفحك، ويمكنك التحدّث إلى Claude Code أو Codex أو أي وكيل يتحدث [ACP (بروتوكول عميل الوكيل)](https://github.com/anthropics/agent-protocol) — وكل ذلك من خلال واجهة متعددة اللوحات تدعم البث المتدفق والتخزين الدائم وإدارة الجلسات:

```bash
npx agentpane
# → http://localhost:6767
```

اختر وكيلًا، وحدد مجلد العمل، ثم ابدأ جلسة. يعمل الوكيل كعملية فرعية على جهازك. لا سحابة، ولا نشر، ولا حسابات.

![شاشة إعداد agentpane مع اختيار الوكيل ومجلد العمل](/blog-images/agentpane-setup.png)

## متعدد اللوحات، متعدد الوكلاء

كان أهم ما أريده هو القدرة على تشغيل عدة وكلاء في وقت واحد ورؤيتهم جنبًا إلى جنب. يوفّر لك agentpane ما يصل إلى أربع لوحات قابلة لتغيير الحجم مع علامات تبويب. يمكنك سحب الجلسات من الشريط الجانبي إلى أي لوحة، أو سحب علامة تبويب لتقسيمها إلى لوحة جديدة. ويظل التخطيط محفوظًا حتى بعد التحديث.

وهذا يعني أنه يمكنك جعل Claude Code يعمل على الواجهة الخلفية في لوحة، بينما يتولى Codex إعادة هيكلة الواجهة الأمامية في لوحة أخرى — على الشاشة نفسها، وفي التطبيق نفسه.

![جلستان تعملان جنبًا إلى جنب — Codex على اليسار، وClaude Code على اليمين](/blog-images/agentpane-hero.png)

## البنية المعمارية

يعمل التطبيق من خلال عمليتين:

* **خادم واجهة برمجة التطبيقات** (Hono، المنفذ 3456) — يُنشئ الوكلاء، ويدير الجلسات، ويتولى التخزين الدائم
* **الواجهة الأمامية للويب** (Next.js، المنفذ 6767) — واجهة المستخدم، مع إعادة توجيه طلبات `/api` إلى الواجهة الخلفية

الواجهة الخلفية مبنية باستخدام [Effect.ts](https://effect.website/)، وتستخدم حقن التبعيات عبر طبقات خدمات قابلة للتركيب:

```
SessionRepo (DB)
  → ConnectionManager (agent subprocess lifecycle)
  → PromptEngine (turn orchestration)
  → EventHub / EventBroadcaster (SSE)
  → WriteQueue (batched DB writes)
```

يجري التواصل بين الوكلاء عبر ACP — ‏JSON-RPC 2.0 عبر stdio. عندما ترسل prompt، ينشئ `PromptEngine` سجلات الجولات، ويمرّر الرسالة إلى العملية الفرعية للوكيل عبر ACP، ثم يعيد بث النتائج عبر Server-Sent Events.

### البث المتدفق مع ضمان سلامة إعادة الاتصال

تشترك الواجهة الأمامية في نقطة نهاية SSE لكل جلسة. ويُسنَد إلى كل حدث معرّف أحادي الزيادة. ويحتفظ `EventBroadcaster` بمخزن دائري (512KB / 1000 حدث)، بحيث إذا انقطع اتصال المتصفح لديك ثم اتصل من جديد، يعيد إرسال الأحداث التي فاتتك باستخدام رأس `Last-Event-ID`. من دون أي فجوات في المحادثة، ومن دون الحاجة إلى تحديث الصفحة يدويًا.

### التخزين الدائم الآمن عند التعطل

تُحفَظ كل الحالة في قاعدة بيانات SQLite محلية في `~/.agentpane/agentpane.db`. لكن عمليات الكتابة لا تحدث مباشرةً — إذ يجمع `WriteQueue` العمليات ويفرّغها على دفعة واحدة كل 50 مللي ثانية. وقبل التفريغ، تُحفَظ العمليات في جدول للتعافي. وإذا تعطّل الخادم أثناء التفريغ، فإنه يتابع من حيث توقّف عند إعادة التشغيل. من دون فقدان أي رسائل.

```sql
-- جدول الاسترداد
CREATE TABLE write_queue_ops (
  id TEXT PRIMARY KEY,
  session_id TEXT NOT NULL,
  op_json TEXT NOT NULL,
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);
```

## ما الذي يراه الوكلاء

لا يهمّ agentpane أيُّ نموذج يشغّل الوكيل — فهو يتحدث ACP. عند الاتصال بجلسة، يقوم `ConnectionManager` بما يلي:

1. يحدّد الملف التنفيذي للوكيل (ويفحص `node_modules/.bin/` لضمان التوافق مع npm)
2. يشغّله كعملية فرعية مع قنوات stdio
3. يتفاوض على إمكانات ACP (المصادقة، التهيئة، الأوضاع، الأوامر)
4. يحاول استئناف الجلسة عند إعادة الاتصال بجلسة سابقة

تتكيّف واجهة المستخدم مع كل ما يدعمه الوكيل. فإذا كان الوكيل يوفّر خيارات تهيئة، فستظهر على شكل قوائم منسدلة. وإذا كان يدعم `slash commands`، فستحصل على إكمال تلقائي. وإذا كان يبث كتل التفكير، فستُعرض كأقسام قابلة للطي.

## الواجهة الأمامية

تطبيق الويب مبنيّ باستخدام Next.js 16 مع React 19 وReact Compiler — من دون أي `useMemo` أو `useCallback` يدوي في أي مكان. تُقسَّم الحالة إلى طبقتين من السياق:

* **SessionProvider** — الجلسة النشطة، فحوصات الحالة، وعمليات CRUD الخاصة بالجلسة
* **LayoutProvider** — إعدادات اللوحات، وترتيب علامات التبويب، والسحب والإفلات

تُدار حالة الخادم (الجلسات، والمحادثات، واستهلاك token) باستخدام TanStack React Query، وتُبطَل صلاحيتها عند ورود أحداث SSE. ويُعرَض Markdown أثناء تدفّقه باستخدام [Streamdown](https://github.com/anthropics/streamdown) مع إبراز الشيفرة نحويًا عبر Shiki.
