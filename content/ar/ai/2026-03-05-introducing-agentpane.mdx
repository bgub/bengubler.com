# التعرّف على agentpane

agentpane هي واجهة ويب محلية لوكلاء البرمجة بالذكاء الاصطناعي. تشغّل `npx agentpane`، فتفتح في متصفحك، ويمكنك التحدث إلى Claude Code أو Codex أو أي وكيل يدعم [ACP (Agent Client Protocol)](https://github.com/anthropics/agent-protocol) — وكل ذلك من خلال واجهة متعددة اللوحات مع البث المباشر، وحفظ الحالة، وإدارة الجلسات:

```bash
npx agentpane
# → http://localhost:6767
```

اختر وكيلاً، واختر مجلد عمل، ثم ابدأ جلسة. يعمل الوكيل كعملية فرعية على جهازك. لا سحابة، ولا نشر، ولا حسابات.

![شاشة إعداد agentpane مع اختيار الوكيل ومجلد العمل](/blog-images/agentpane-setup.png)


## متعدد الألواح، متعدد الوكلاء

كان أهم ما أريده هو إمكانية تشغيل عدة وكلاء في الوقت نفسه ورؤيتهم جنبًا إلى جنب. يوفّر لك agentpane ما يصل إلى أربع لوحات قابلة لتغيير الحجم مع علامات تبويب. يمكنك سحب الجلسات من الشريط الجانبي إلى أي لوحة، أو سحب علامة تبويب لتقسيمها إلى لوحة جديدة. ويبقى التخطيط محفوظًا حتى بعد تحديث الصفحة.

هذا يعني أنه يمكنك جعل Claude Code يعمل على الواجهة الخلفية في لوحة، وCodex يعيد هيكلة الواجهة الأمامية في لوحة أخرى — على الشاشة نفسها، وفي التطبيق نفسه.

![جلستان تعملان جنبًا إلى جنب — Codex على اليسار، وClaude Code على اليمين](/blog-images/agentpane-hero.png)

## البنية

يتكوّن التطبيق من عمليتين:

* **خادم API** (Hono، المنفذ 3456) — يُنشئ الوكلاء، ويدير الجلسات، ويتعامل مع التخزين الدائم
* **الواجهة الأمامية للويب** (Next.js، المنفذ 6767) — واجهة المستخدم، مع إعادة كتابة استدعاءات `/api` لتُوجَّه إلى الواجهة الخلفية

تم بناء الواجهة الخلفية باستخدام [Effect.ts](https://effect.website/) وتستخدم حقن التبعيات عبر طبقات خدمات قابلة للتركيب:

```
SessionRepo (DB)
  → ConnectionManager (agent subprocess lifecycle)
  → PromptEngine (turn orchestration)
  → EventHub / EventBroadcaster (SSE)
  → WriteQueue (batched DB writes)
```

يتواصل الوكلاء عبر ACP — ‏JSON-RPC 2.0 عبر stdio. عند إرسال موجّه، ينشئ `PromptEngine` سجلات الجولات، ويمرّر الرسالة إلى العملية الفرعية للوكيل عبر ACP، ثم يدفّق النتائج مجددًا عبر Server-Sent Events.


### البث مع أمان عند إعادة الاتصال

تشترك الواجهة الأمامية في نقطة نهاية SSE لكل جلسة. ويحصل كل حدث على معرّف تزايدي. ويحتفظ `EventBroadcaster` بمخزن دائري (512KB / 1000 حدث) بحيث إذا انقطع اتصال المتصفح ثم أُعيد الاتصال، يعيد إرسال الأحداث التي فاتتك باستخدام ترويسة `Last-Event-ID`. لا فجوات في المحادثة، ولا حاجة إلى تحديث الصفحة يدويًا.

### حفظ البيانات بأمان عند التعطل

تُحفَظ جميع الحالات في قاعدة بيانات SQLite محلية في `~/.agentpane/agentpane.db`. لكن عمليات الكتابة لا تتم مباشرةً — بل يقوم `WriteQueue` بتجميع العمليات وكتابتها على دفعات كل 50 مللي ثانية. وقبل تنفيذ الكتابة، تُحفَظ العمليات في جدول للاسترداد. وإذا تعطّل الخادم أثناء الكتابة على الدفعات، فإنه يستأنف من حيث توقّف عند إعادة التشغيل. لا يحدث أي فقدان للرسائل.

```sql
-- جدول الاسترداد
CREATE TABLE write_queue_ops (
  id TEXT PRIMARY KEY,
  session_id TEXT NOT NULL,
  op_json TEXT NOT NULL,
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);
```


## ما يراه الوكلاء

لا يهم agentpane أيُّ نموذج يشغّل الوكيل — فهو يتواصل عبر ACP. عند الاتصال بجلسة، يقوم `ConnectionManager` بما يلي:

1. يحدّد الملف التنفيذي للوكيل (ويبحث في `node_modules/.bin/` للتوافق مع npm)
2. يشغّله كعملية فرعية باستخدام أنابيب stdio
3. يتفاوض على إمكانات ACP (المصادقة، والإعدادات، والأوضاع، والأوامر)
4. يحاول استئناف الجلسة عند إعادة الاتصال بجلسة سابقة

تتكيّف واجهة المستخدم مع كل ما يدعمه الوكيل. إذا كان الوكيل يوفّر خيارات إعداد، فستظهر على شكل قوائم منسدلة. وإذا كان يدعم أوامر الشرطة المائلة، فستحصل على إكمال تلقائي. وإذا كان يبث كتل التفكير، فستُعرض كأقسام قابلة للطي.

## الواجهة الأمامية

تطبيق الويب مبنيّ باستخدام Next.js 16 مع React 19 وReact Compiler — من دون أي استخدام يدوي لـ `useMemo` أو `useCallback` على الإطلاق. وتنقسم الحالة إلى طبقتين من السياق:

* **SessionProvider** — الجلسة النشطة، فحوصات الحالة، عمليات CRUD للجلسة
* **LayoutProvider** — إعدادات اللوحات، ترتيب علامات التبويب، السحب والإفلات

تُدار حالة الخادم (الجلسات، المحادثات، استخدام التوكنات) باستخدام TanStack React Query، وتُبطَل صلاحيتها عند ورود أحداث SSE. ويُعرَض Markdown أثناء تدفّقه باستخدام [Streamdown](https://github.com/anthropics/streamdown) مع تمييز الشيفرة بواسطة Shiki.
